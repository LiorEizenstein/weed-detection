Weed-Centering Control Loop

The arm scans the field, locks onto a detected weed using a pixel-error feedback loop, and fires the laser once the weed is centred in the camera frame.

Parameters & Design Notes

Control Loop Parameters

ParameterValueRole
FIRE_ZONE_FRAC0.20Dead-zone threshold — fires when weed is within 20% of image centre in both axes
PAN_GAIN0.10 rad/pxCorrection step size for X-axis (shoulder_pan)
WRIST2_GAIN0.10 rad/pxCorrection step size for Y-axis (wrist_2)
WEED_MOVE_TIME1.0 sDuration of each correction move
WEED_LOCK_TIMEOUT8.0 sGive up on lost weed after this long
TREATED_ZONE_RAD0.25 radBlacklist radius around a fired pan angle

Why single-axis correction per tick?

Rotating shoulder_pan shifts the image in both X and Y simultaneously, so correcting both axes at once creates a coupled feedback loop that can diverge.

Correcting only the dominant error axis per step decouples the loop and ensures the arm spirals in on the target instead of oscillating around it.
